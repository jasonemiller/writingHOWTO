    • 1 - clone project from github to local repo
      To obtain the project materials, you must use git and create a clone of the master repository on your work machine. This means you’ll need git installed and you’ll need an active internet connection.

    • 3 - stage changes (git add)

      • can check with 'git status'

      • stage files with changes to be included in 'commit'

    • 4 - commit changes (git commit)

    • 5 - fetch new files from github repo

    • 6- merge new files with your version; you will now have the most up-to-date version of the project as your local repo

      • if there are not file/text conflicts, you can proceed to the next step

      • if the merge fails with a 'conflict' message (and file name), you'll need to do some work

        • open the file in an editor and resolve the conflict (look for a series of '<' at the start of the conflict and a series of '>' at the end of the conflict. The conflicting text is between them

        • replace conflicted text with your fix

        • restage the edited file with 'git add [filename]'

        • return to step 4

    • 7 - push changes to common repo at github
